# Security Policy

## Reporting a vulnerability

Please report security vulnerabilities through **GitHub Private Vulnerability Reporting** (GitHub Security Advisories) for this repository.

- Do **not** open public issues for security problems.
- Include clear reproduction steps, affected files/components, and impact.
- If possible, include a proof of concept and proposed mitigation.

## What to expect

- We will acknowledge valid reports and triage them as quickly as possible.
- We will investigate, patch, and coordinate disclosure once a fix is available.

## Scope

This policy covers the code in this repository and release artifacts produced from it.

## Out of scope

- Social engineering or phishing attempts
- Vulnerabilities requiring physical access to a user's machine
- Issues only affecting unsupported forks or heavily modified deployments
