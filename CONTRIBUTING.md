# Contributing to TrenchersPaperTrading

Thanks for your interest in improving this project.

## Ground rules

- External contributions are accepted through **Pull Requests only**.
- Direct pushes to `main` are restricted to the repository owner/maintainer.
- Keep PRs focused, small, and scoped to one change.

## Workflow

1. Fork the repository.
2. Create a feature branch from `main`.
3. Make your changes and update related docs when behavior changes.
4. Open a Pull Request against `main` with a clear description and test steps.

## Commit format

This repository uses semantic-release. Use Conventional Commits when possible, for example:

- `feat: add ...`
- `fix: correct ...`
- `chore: update ...`

## Pull request expectations

- Explain **what** changed and **why**.
- Include screenshots for UI changes.
- Avoid unrelated refactors in the same PR.
- Address review feedback in follow-up commits.
